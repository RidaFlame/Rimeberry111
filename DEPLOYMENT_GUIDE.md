## üöÄ Deployment Steps

### Method 1: Deploy to gh-pages Branch (Recommended)

1. **Build your website:**
   ```bash
   npm run build
   ```
   This creates a `dist` folder with all your website files.

2. **Push dist folder to gh-pages branch:**
   
   **Option A: Using git subtree (Simple)**
   ```bash
   # Navigate to your project directory
   cd "C:\Users\Acer1\Desktop\swaray website"
   
   # Build the project
   npm run build
   
   # Add dist folder to git (if not already tracked)
   git add dist
   git commit -m "Add dist folder for deployment"
   
   # Push to gh-pages branch (this command creates the branch if it doesn't exist)
   git subtree push --prefix dist origin gh-pages
   ```
   
   **Option B: Using separate gh-pages branch (Manual)**
   ```bash
   # Build the project
   npm run build
   
   # Create and switch to gh-pages branch
   git checkout --orphan gh-pages
   
   # Remove all files except dist
   git rm -rf .
   
   # Copy contents of dist to root
   # On Windows PowerShell:
   Copy-Item -Path "dist\*" -Destination "." -Recurse -Force
   
   # Stage all files
   git add .
   
   # Commit
   git commit -m "Deploy to GitHub Pages"
   
   # Push to gh-pages branch
   git push origin gh-pages --force
   
   # Switch back to main branch
   git checkout main
   ```

### Method 2: Deploy from docs folder

1. **Update vite.config.js** to build to `docs` folder:
   ```javascript
   build: {
     outDir: 'docs',
     // ... rest of config
   }
   ```

2. **Build and push:**
   ```bash
   npm run build
   git add docs
   git commit -m "Deploy to GitHub Pages"
   git push origin main
   ```

## üìã GitHub Pages Setup on GitHub

1. Go to your repository: `https://github.com/RidaFlame/Rimeberry111`
2. Click **Settings** ‚Üí **Pages** (in left sidebar)
3. Under **Source**, select **"Deploy from a branch"**
4. Select **Branch**: `gh-pages` (or `main` if using docs folder)
5. Select **Folder**: `/ (root)` (or `/docs` if using docs folder)
6. Click **Save**

## üîß Important Configuration Notes

### Base Path
The `vite.config.js` is configured with:
```javascript
base: '/Rimeberry111/'
```

This is **correct** for a project repository. If you change the repository name, update this path accordingly.

### .nojekyll File
The `.nojekyll` file is in the `public` folder and will be automatically copied to the build output. This prevents GitHub from processing files with Jekyll, which fixes the white screen issue.

### HashRouter
Your site uses HashRouter, which works perfectly with GitHub Pages:
- URLs will look like: `https://ridflame.github.io/Rimeberry111/#/about`
- This works regardless of base path issues

## üêõ Troubleshooting

### White Screen Issue

**Causes:**
1. Missing `.nojekyll` file ‚Üí ‚úÖ Already in public folder
2. Incorrect base path ‚Üí ‚úÖ Configured as `/Rimeberry111/`
3. Missing index.html ‚Üí ‚úÖ Generated by build
4. 404 errors on assets ‚Üí Check browser console (F12)

**Solution:**
1. Open browser console (F12) and check for 404 errors
2. Verify the base path in `vite.config.js` matches your repository name
3. Ensure `.nojekyll` is in the deployed folder
4. Check that all files were pushed correctly

### Assets Not Loading (404 errors)

**Check:**
1. Browser console (F12) ‚Üí Network tab
2. Look for failed requests (status 404)
3. Verify paths start with `/Rimeberry111/assets/...`

**Fix:**
- Ensure base path in `vite.config.js` is `/Rimeberry111/`
- Rebuild: `npm run build`
- Redeploy to gh-pages branch

### Deployment Fails

**Check:**
1. GitHub repository Settings ‚Üí Pages ‚Üí Confirm source is set to "Deploy from a branch"
2. Verify gh-pages branch exists and has files
3. Check that index.html exists in gh-pages branch root

### Site Not Updating

**Solution:**
- GitHub Pages can take 1-2 minutes to update
- Hard refresh browser: Ctrl+F5 (Windows) or Cmd+Shift+R (Mac)
- Check GitHub repository Settings ‚Üí Pages for deployment status
